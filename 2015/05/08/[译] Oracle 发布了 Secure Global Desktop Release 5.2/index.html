[译] Oracle 发布了 Secure Global Desktop Release 5.2

  • em12c

  • sso
  • 安全访问
  • 应用程序虚拟化
虚拟化

来源: http://www.dbta.com/Editorial/News-Flashes/Oracle-Rolls-Out-Secure-Global-Desktop-Release-52-103730.aspx

Oracle已经发布了Oracle Secure Global Desktop (SGD) Release 5.2，作为该公司的远程访问解决方案，为Windows，Linux，Solaris以及大型主机等服务器端托管的企业级应用程序和桌面提供安全的远程访问。

SGD让用户能够使用任何设备，在任何地方都可以安全地工作，同时为运行在数据中心的应用程序和桌面环境提供管理员可控的访问，在BYOD的世界中为企业提供了非常关键的优势。

SGD可以工作在流行的客户端设备上，包括Windows PC、Macs、Linux PC以及诸如Apple iPad和基于Android的平板设备。

尽管其还是以原来的“Tarantella”名字被人熟知，该产品的名称在2005年被Sun Microsystems收购时改成了Sun Secure Global Desktop，之后在Oracle收购Sun之后又被更名为Oracle Secure Global Desktop。

“其实在收购之前，Oracle就一直是该产品的最大用户”，Wim Coekaerts，Linux和虚拟化工程高级副总裁。在Tarantella时代，Oracle就开始使用它了。“我们将其用在开发环境中，为远程开发者提供托管环境的访问，同时远程用户也通过它来访问集中管理的应用程序环境”，他如是说。

另外，Oracle本身就有成百上千的的应用程序，这些应用程序大多数是基于Web技术的，Coekaerts提到。当不同版本的应用程序对浏览器的需求也不同的时候，SGD会带来显著的优势。

“可以是任何类型的客户端，因为它是基于浏览器的。它使用了HTML5，因此你可以使用iPad，或者Android设备，或者你的Mac - 这真的不重要”，Coekaerts说，“应用程序的使用体验就和本地应用一模一样，而且可以在任何网络环境下使用”，而且，从安全性的角度来说，因为本地没有存储数据，所以这是一种访问企业数据的非常安全的方法。

此外，说到它能提供的功能，Coekaerts说，用户可能会使用某个版本的应用程序，而该应用程序需要Internet Explorer 6，同时他还可能有一个新版本的应用程序需要IE7，可问题是在一个桌面环境中只能运行一个版本的IE。但是，在托管环境中，可以安装多个不同版本的浏览器。“这是一个典型的例子，证明了在真实世界中的客户环境中SGD非常有用。”

SGD Release 5.2提供一些重要的功能强化，Coekaerts提到。

与Oracle Access Manager集成的单点登录(Single sign-on) 能够对应用程序实现即时访问，而用户只需要输入一次身份信息。“可以这么说，我们实际上从一进门开始就集成了单点登录”，Coekaerts说。Oracle Secure Global Desktop与Oracle Access Manager的集成可以使用Oracle Access Manager的单点登录的功能来认证SGD服务器的用户，和从SGD工作区打开的远程应用程序的用户。单点登录使SGD用户在访问多个应用程序时只需要登录一次，简化了密码管理。

利用 Oracle Enterprise Manager 监视，在5.2发行版中通过插件来实现的，在Oracle Enterprise Manager中对多个SGD部署环境进行集中的监视。“Oracle Enterprise Manager是Oracle的一站式管理套件，它能够管理数据库，中间件，应用程序，存储设备，Linux等等。Oracle的所有产品都会为这个集中管理工具提供一个插件，这就意味着在监视和管理所有不同的组件时，你只需要在这一个工具中加入一个插件即可”，Coekaerts说，“在5.2中，我们现在支持将SGD作为Enterprise Manager的一个组件或者一个插件”。

通过这些插件，可以监视和分析SGD服务器以及Gateway服务器的健康和性能状态。对于SGD服务器，该插件收集各种性能和配置指标，诸如阵列状态，用户和应用程序，数据存储对象，SGD服务器进程，版本以及补丁等。对于Gateway服务器，各种性能和配置指标也会被收集，包括服务中的连接信息，注册的SGD服务器，版本，补丁等。这些插件收集的指标数据可以被一些报表工具，如Oracle Business Intelligence Publisher进行进一步的分析和处理。

在新发行版中的第三个组件是增强的基于角色的用户管理机能，Coekaerts说，随着云计算和托管环境的不断增长，这是越来越重要的功能。“传统上，一个产品只需要一个管理员来管理所有不同的用户”。但是，Oracle已经将管理员也分成不同的水平和权限。在这一版本中引入的新命令也简化了SGD部署中软件更新（补丁）的安装过程。除了使用命令行，图形化界面的管理控制台可以用于添加SGD服务器到阵列中，同时使用安全的阵列内通信，“在云环境中，在托管主机服务环境中，这会是一个很不错的加分点”。

新版本中的PulseAudio集成可以对使用PulseAudio的应用程序实现音频重定向，大大增加了SGD支持的音频应用程序范围。此外，新增加的对UNIX音频输入的支持，使之与对Windows的音频支持处于同一水平线上，而且也可以为与OSS,SADA或PulseAudio兼容的应用程序使用音频输入。

除了新特性和更广泛的服务器，客户端，以及浏览器支持之外，本次发行版也提供了增强的安全性和性能，在各种网络环境下提供更丰富的用户体验。

Oracle Secure Global Desktop 5.2秉持其一贯的简洁性同时更紧密地与Oracle产品，技术以及解决方案进行了集成，如Oracle Managed Cloud Services，Oracle Enterprise Manager，和Oracle Access Manager。
